# 查询管理系统 MVP 功能清单

## 1. 文档目标

本文档用于把 PRD 和技术方案进一步拆解成可执行的 MVP 功能项，方便后续按阶段开发。

原则：

- 先做查询，不做复杂录入
- 先做可用，再做精细
- 先做文档驱动，再做更强结构化

## 2. MVP 范围总览

MVP 阶段目标：

1. 能读取当前项目资料
2. 能建立基础索引
3. 能进行全文搜索
4. 能查看总览、时间线、指标、文档详情
5. 能跳转回原始文件

## 3. 功能模块拆解

### 3.1 文件扫描与索引

#### 必做

- 扫描项目根目录
- 识别 markdown、图片、PDF
- 识别主文档、住院文档、报告图片
- 记录文件路径、修改时间、大小、类型

#### 产出

- 文件清单
- 可供后端查询的索引缓存

### 3.2 markdown 文档解析

#### 必做

- 解析主文档
- 解析住院整理文档
- 提取标题、段落、时间点、图片链接

#### 建议做

- 提取“当前用药”片段
- 提取“近期发作情况”时间轴片段
- 提取“住院概况”片段

### 3.3 结构化抽取

#### 必做

- 抽取事件日期
- 抽取事件类型
- 抽取药物名称
- 抽取检查名称
- 抽取结果文本

#### 建议做

- 抽取剂量
- 抽取数值型指标
- 抽取单位
- 抽取是否住院
- 抽取是否 OCR 近似值

### 3.4 全文搜索

#### 必做

- 支持关键词搜索
- 返回命中文档
- 返回命中片段
- 支持跳转到相关文档

#### 查询示例

- 呕吐
- 地西泮
- 吡仑帕奈
- 同型半胱氨酸
- 住院

### 3.5 总览页

#### 必做

- 展示患者基础信息
- 展示主要诊断
- 展示当前用药
- 展示最近一次发作
- 展示最近一次住院
- 展示长期重点提示

### 3.6 时间线页

#### 必做

- 展示发作、住院、调药、检查等事件
- 支持按事件类型筛选
- 支持按时间倒序查看

#### 建议做

- 支持按年份筛选
- 支持点击后展开详情抽屉

### 3.7 检查结果页

#### 必做

- 按检查类型查看结果列表
- 显示日期、结果、单位、说明
- 支持跳转到原始报告图

#### 建议做

- 同型半胱氨酸历史列表
- 吡仑帕奈血药浓度历史列表
- 血氨 / 乳酸 / 肝肾功等分组浏览

### 3.8 文档详情页

#### 必做

- 展示主文档内容
- 展示住院文档内容
- 保留图片和链接
- 支持 markdown 渲染

#### 技术约束

- 前端 markdown 展示优先使用 `@ant-design/x-markdown`

### 3.9 原始资料页

#### 必做

- 按文件夹或文件类型浏览原始资料
- 支持查看报告图片
- 支持跳转 PDF

### 3.10 图表能力

#### MVP 可做

- 同型半胱氨酸趋势图
- 吡仑帕奈血药浓度趋势图
- 发作 / 住院次数简要统计图

#### 原则

- 图表是辅助说明
- 所有图表必须能回到文本和原始文件

## 4. 页面优先级

### P0

- 总览页
- 全文搜索
- 时间线页
- 文档详情页

### P1

- 检查结果页
- 原始资料页
- 指标趋势图

### P2

- 高级筛选
- 标签系统
- 更复杂统计面板

## 5. 后端任务拆解

### P0

- 建立项目扫描器
- 建立 SQLite 数据库
- 建立全文索引
- 建立主文档/住院文档解析器
- 提供基础查询 API

### P1

- 增强结构化抽取
- 增强指标聚合
- 增加趋势接口

## 6. 前端任务拆解

### P0

- 初始化 React + Vite + Ant Design + Ant Design X 项目
- 建立应用框架 Layout
- 建立路由
- 建立总览页
- 建立搜索页
- 建立时间线页
- 建立文档详情页

### P1

- 建立检查结果页
- 建立基于 `@ant-design/plots` 的图表展示组件
- 建立原始资料浏览页

## 7. MVP 验收标准

### 验收项 1

输入关键词后，能返回相关文档与命中片段。

### 验收项 2

能在系统里看到当前主文档概览，而不必手动打开 markdown。

### 验收项 3

能查看发作 / 住院 / 调药的时间线。

### 验收项 4

能查看重点指标历史结果列表。

### 验收项 5

能从结果跳回原始整理稿或原始报告文件。

## 8. 当前推荐开发顺序

建议开发顺序：

1. 建后端索引与搜索
2. 建前端基础框架
3. 做总览页与搜索页
4. 做时间线页
5. 做文档详情页
6. 做检查结果页
7. 最后补图表

这个顺序最稳，也最容易尽快得到一个可用版本。
