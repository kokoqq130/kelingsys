# 查询管理系统页面信息架构

## 1. 文档目标

本文档用于定义查询管理系统的页面结构、导航方式、页面职责和信息组织方式。

目标是让系统满足：

- 好找
- 好查
- 好回溯
- 桌面端清晰
- 移动端可用

## 2. 总体导航结构

建议左侧主导航如下：

1. 总览
2. 时间线
3. 检查结果
4. 文档资料
5. 搜索

可选补充：

6. 系统状态

## 3. 页面结构说明

### 3.1 总览页

#### 页面目标

让用户一打开系统，就能知道当前最重要的信息。

#### 页面内容

- 患者基础信息卡片
- 当前主要诊断
- 当前用药方案
- 最近一次发作摘要
- 最近一次住院摘要
- 长期重点提示
- 快捷入口

#### 建议组件

- `Layout`
- `Card`
- `Descriptions`
- `Tag`
- `Alert`
- `Statistic`

### 3.2 时间线页

#### 页面目标

按时间查看发作、住院、调药、检查等事件，快速建立时间脉络。

#### 页面内容

- 时间线主列表
- 事件类型筛选
- 日期范围筛选
- 事件详情抽屉

#### 单条事件应显示

- 日期
- 事件类型
- 标题摘要
- 关键补充信息
- 对应文档链接

#### 建议组件

- `Timeline`
- `Card`
- `Tag`
- `Select`
- `DatePicker`
- `Drawer`

### 3.3 检查结果页

#### 页面目标

按检查类型查询历史结果，避免反复翻文档。

#### 页面结构

- 左侧：检查类型列表或分组
- 右侧：结果列表 / 趋势图 / 原始文件入口

#### 页面内容

- 检查项目名称
- 日期
- 结果文本
- 单位
- 是否异常
- 来源说明
- 原图链接
- 文档链接

#### 建议组件

- `Tabs`
- `Table`
- `Segmented`
- `Card`
- `Tag`
- 图表组件

### 3.4 文档资料页

#### 页面目标

浏览和回溯主文档、住院文档、报告索引、原始资料。

#### 页面结构

- 左侧：文件树 / 文件夹结构
- 右侧：文档预览区

#### 页面内容

- 主文档
- 各次住院整理文档
- 报告索引
- 原始图片 / PDF

#### markdown 展示要求

- 默认使用 `@ant-design/x-markdown` 渲染 markdown
- 不把通用 markdown 渲染库作为首版默认主方案，除非后续明确存在能力缺口

#### 建议组件

- `Tree`
- `Table`
- `Drawer`
- `Tabs`
- `XMarkdown`

### 3.5 搜索页

#### 页面目标

支持直接通过关键词查所有资料。

#### 页面结构

- 顶部：搜索框
- 左侧：筛选条件
- 右侧：结果列表

#### 支持筛选

- 文件类型
- 事件类型
- 日期范围
- 是否住院相关

#### 结果展示

- 标题
- 命中片段
- 所属文档
- 日期
- 跳转按钮

#### 建议组件

- `Input.Search`
- `Form`
- `Select`
- `List`
- `Table`
- `Highlight` 或自定义关键词高亮

## 4. 移动端信息架构

### 4.1 原则

- 桌面端是主形态
- 移动端不追求复杂后台体验
- 但核心查询流程必须可用

### 4.2 移动端保留能力

- 查看总览
- 搜索
- 查看时间线
- 查看住院文档
- 查看当前用药
- 查看重点指标

### 4.3 移动端布局建议

- 左侧导航改为顶部或抽屉菜单
- 多栏布局改为单栏
- 大表格优先降级成卡片列表
- 筛选区优先放在抽屉中

## 5. 典型页面跳转关系

### 路径 1：从总览进入

总览 -> 最近一次住院 -> 住院整理详情 -> 原始检查图

### 路径 2：从搜索进入

搜索 -> 命中文档 -> 命中片段详情 -> 原始文档 / 原图

### 路径 3：从检查结果进入

检查结果页 -> 某个指标 -> 历史结果 -> 某次结果详情 -> 原图

### 路径 4：从时间线进入

时间线 -> 某次发作 -> 调药详情 / 住院详情 / 检查详情

## 6. 首页卡片建议

总览页可考虑以下卡片：

- 患者基本信息
- 当前诊断
- 当前治疗方案
- 最近一次发作
- 最近一次住院
- 长期重点指标提示

## 7. 图表布局建议

建议只在确实有趋势意义的页面加入图表：

- 检查结果页顶部：趋势图
- 时间线页顶部：年度事件分布
- 总览页：简单统计卡片

图表生态建议：

- 默认优先使用 `@ant-design/plots`
- 只有复杂场景再补充其他图表库

不要把图表铺满所有页面。

## 8. MVP 最小页面集合

如果只做最小版本，建议先上线以下页面：

1. 总览页
2. 搜索页
3. 时间线页
4. 文档详情页
5. 检查结果页

这样就已经足以覆盖“快速了解当前情况”和“快速查历史资料”两种核心需求。
