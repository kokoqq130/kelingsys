# 项目规则

## 适用范围

- 当前工作区使用项目本地 skill，目录为 `[.codex/skills](./.codex/skills)`。
- 优先使用项目本地 skill，不使用同名的用户级或全局 skill。
- 如果项目流程发生了会影响重复性工作的变化，需要在同一轮修改里同时更新 `AGENTS.md` 和对应的本地 skill，避免两边规则漂移。

## 主文档

- 面向医生的主文档是 `[柯灵基本信息.md](./柯灵用/柯灵基本信息.md)`。
- 主文档中保留当前准确的用药剂量。
- 当前用药方案变化时，既要更新主文档里的当前剂量，也要保证前面的调药时间轴仍然能解释为什么会调整到现在的方案。
- 更新“目前治疗方案”时，尽量保留明确的“截至日期”或“按某次住院时整理”的说明。
- 对慢性病后续长期参考价值较高的监测指标，可以保留在主文档中。
- 只对某一次住院有意义、后续参考价值不高的短期化验结果，不要大量堆进主文档。

## 住院文档

- 每次住院或当前状态相关的时效性内容，放到对应的住院/检查报告文件夹里单独整理成 markdown。
- 每份住院整理应包含：事件经过、主要症状、即时处理、分组化验结果解读、原始报告索引。
- 通过 OCR 得到但不能完全确认的数值，要明确写成“约”，并保留原图链接。

## OCR 与报告文件

- 先 OCR，再给 `微信图片_*.jpg`、`微信图片_*.png` 这类泛文件名改名。
- 只有在日期和报告类型基本确认后，才重命名报告文件。
- 报告文件名优先采用 `YYYY-MM-DD_报告名称.ext` 这种格式。
- 如果日期或数值不确定，不要假装是精确值。

## Skill 同步

- 本地医疗整理 skill 是 `[medical-report-organizer](./.codex/skills/medical-report-organizer)`。
- 本地主文档 PDF 导出 skill 是 `[main-doc-pdf-export](./.codex/skills/main-doc-pdf-export)`。
- 如果文档结构、命名规则、整理逻辑、解释习惯发生变化，要同步更新对应本地 skill。

## PDF 规则

- 修改主 markdown 文档后，不代表必须立刻重建 PDF。
- 修改 `[柯灵基本信息.md](./柯灵用/柯灵基本信息.md)` 后，需要询问用户是否现在刷新对应 PDF。
- 只有在用户明确确认后，才生成或覆盖主文档 PDF。
- 主文档 PDF 默认输出到 `[柯灵基本信息.pdf](./柯灵用/柯灵基本信息.pdf)`，除非用户另外指定路径。
